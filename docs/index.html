kymera

A JAX-native multi-agent grid simulator library for communication-constrained swarm research. Like gymnax, but multi-agent — you write the training loop; kymera gives you the env, the rollout primitive, the metrics, and the viz.

kymera is the successor to zymera, rebuilt around composition: an environment is five swappable components (world generation, dynamics, communication, observation, mission) snapped onto one orchestrator. A new research idea is a new component — usually a plain function written in your experiment script — never an edit to the simulator.

Parity kymera reproduces zymera v0 exactly: with the same PRNGKey and policy, walls, spawns, actions, positions, gossip belief maps, and observations are bit-for-bit identical, and per-step rewards match to 1e-5. The gate lives in tests/test_parity.py against golden trajectories dumped from the live v0 install.

Install

python -m venv .venv && source .venv/bin/activate
pip install -e ".[viz,dev]"     # core is headless; viz pulls matplotlib/pillow

pytest tests/ -q                # full suite incl. the v0 parity gate

import kymera never pulls plotting or training libraries — the core stays jit/vmap-clean and import-fast. Drawing lives in kymera.viz, opt-in.

Quick start

import jax, kymera
from kymera import viz

env  = kymera.make("comm-coverage", grid=16, n_agents=4, comm_r=5)
traj = kymera.rollout(env, kymera.random_policy, 70, jax.random.PRNGKey(0),
                      keep="all", collect=("reward_terms",))
viz.make_report(traj, "report.html", env=env)

Every env is a deterministic function of its spec and the key — re-run a rollout to re-render anything; never archive heavy trajectories.

The five components

One GridEnv.step runs: dynamics → visit counts → cover footprint → channel delivery (comm_graph := delivered edges) → metrics.derive (shared quantities computed once) → mission update / reward / done → observation.

componentownsbuilt-ins
WorldGenwalls + spawn positions open grid, random walls, ASCII map files, rooms; scatter / clustered / fixed spawns
Dynamicsmovement, collision, action masks boundary-clip + wall-revert; sequential-claim hard collisions with a blocked signal; collision-free masked sampling for trainers
CommChannelwho hears what, when disk topology; gossip with payload ring buffer (delay), per-edge dropout; delivered edges wired into World.comm_graph
ObsBuilderwhat policies see vector obs; named-channel grid obs (belief, positions, neighbors, local frontier) + a central critic view
Missionreward, termination, metrics, overlays sum of named RewardTerms with per-term logging; PBRS and soft-CBF combinators; per-group missions for red-within-blue

Doctrine

Rules inherited from the RedWithinBlue / zymera research program, baked in so they can't be forgotten:

Doctrine Eval samples π — never argmax. Argmax collapses the symmetry-breaking policies use for who-goes-where tie-breaks.
Doctrine Multi-seed via vmap, never a Python loop. Episodes are fixed-length lax.scan; sequential conflicts resolve via scan-over-agents.
Doctrine The reward term set is static. Weights may become traced later; never gate compilation on w == 0.
Doctrine Connectivity reads potential topology unless a term opts into delivered edges; both are logged. Decided before the dropout era so results stay comparable across it.
Doctrine The reset/step key-split order is frozen. Reproducibility and the v0 parity gate ride on it.
